Modular Analysis (MA) enables users to develop interactive web apps from Jupyter notebooks and deploy them to users in their Ganymede tenant.  These applications are natively integrated with Ganymede data storage, enabling them to present context-rich interfaces for data capture and analyses.

## Accessing Modular Analysis

To access MA, click on the <div className="button darkblue_button"><DotChartOutlined /> **Analysis**</div> icon in the left sidebar.

<div style={{ textAlign: 'center' }}>
    <img
        alt="Bioprocess and CDS Modular Analysis demo apps"
        src="https://ganymede-bio.mo.cloudinary.net/ma/NavigatingToModularAnalysis_20250403.png"
        width="80%"
    />
</div>

Each MA application receives its own tile, containing the name, description, and icon for the app, as well as the following buttons:

- <div className="button lightblue_noborder_button"><PlayCircleOutlined /></div>: launch the MA application
- <div className="button lightblue_noborder_button"><CodeOutlined /></div>: edit the code behind an MA application in a Jupyter notebook
- <div className="button lightblue_noborder_button"><EditOutlined /></div>: edit the name, description, and or icon on the tile for the app

New MA Applications can be created by clicking the <div className="button darkblue_outline_button"><PlusOutlined /> New</div> icon.



## Base Applications

Ganymede offers a set of base applications, which could then be modified and adapted for your workflows.  

For example, the base bioprocess MA application could be configured with campaign and run information specific to your labs, drawing from your LIMS or metadata derived from relevant instruments.
